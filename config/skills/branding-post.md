# スキル: ブランディング投稿 (URL無し)

## 目的
X API の URL 付き投稿単価($0.20)を回避し、通常単価($0.015)で配信する「URL無しブランディング投稿」の生成ルール。フォロワー基盤を作るための共感・知見・警告投稿。

## コスト背景
- URL無し投稿: **$0.015/件** (URL付きの 1/13 のコスト)
- 月60投稿のうち **30件を URL無し** にすることで月コストを $12 → $6.5 に圧縮
- セミナー集客投稿([seminar-promotion.md](seminar-promotion.md))と **交互にローテーション**

## ローテーション方針
- **偶数日 (2, 4, 6...)**: URL無し(branding-post) = 本スキル
- **奇数日 (1, 3, 5...)**: URL付き(seminar-promotion) = 集客

毎回生成時に `date +%d` で偶奇確認し、どちらのスキルを使うか自動選択する。

## 投稿パターン(CLAUDE.mdの4類型に準拠)

本スキルでは **URLを一切含めない**。代わりに「共感」「知見の開示」「問いかけ」で余韻を残す。

### パターンA: 1行Tips/警告型 (推奨・C型準拠)
- **構造**: 40〜120字で完結、警告or共感or気づきの1トーン
- **例**:
  ```
  Claudeを本気で使う人が口を揃える共通点:
  「最初の3日は雑に投げる、そこから精度を上げる」
  
  完璧なプロンプトを最初から書こうとするほど、
  逆に学習が遅くなるという罠。
  ```

### パターンB: 実践知見型 (B型準拠・URL無し版)
- **構造**: ノウハウの3点箇条書き、URL ではなく「続きは経験談」で締める
- **例**:
  ```
  メルカリがClaude導入で変えた3つのこと
  
  ①設計レビューをClaudeに壁打ち→論点整理
  ②コード生成は"出力後に削る"前提で雑に依頼
  ③ドキュメント生成は最終工程ではなく設計前にやる
  
  「AIに仕事を任せる」ではなく「AIと会話しながら考える」が正解らしい。
  ```

### パターンC: 事件/スクープ型 (A型準拠・URL無し版)
- **構造**: 業界ニュース反応、CTA の代わりに「問いかけ」or「体言止め」
- **例**:
  ```
  【速報】Anthropic、評価額が$350Bに到達。
  ChatGPT運営の半額まで肉薄。
  
  ・16ヶ月で売上30倍
  ・Claude Codeだけで$2.5B
  ・大手が"乗り換え"フェーズへ
  
  「ChatGPT止まり」が最大の機会損失になる時代。
  ```

### パターンD: 保存版まとめ型 (D型準拠・URL無し版)
- **構造**: ツール/機能の短い箇条書き、「スレッドで詳細」or「コメントで質問歓迎」で締め
- URL の代わりに自分の体験コメントを最後に置く

---

## ✅ 必ず守るルール

1. **URLを一切含めない**(http/https/t.co すべて禁止)
2. ハッシュタグも使わない(CLAUDE.md 準拠)
3. 字数: 40〜220字(URL無しなのでコンパクトに)
4. 最後は「問いかけ」「体言止め」「余韻」のいずれか
5. 商品・セミナー名を直接出さない(間接的な認知のみ)

## 🚫 避けること

- セミナーURL、LP、個別の商品誘導 → [seminar-promotion.md](seminar-promotion.md) 側で実施
- 「詳しくは○○」系の外部誘導
- ハッシュタグ
- 数字スタート

## 採用語彙(URL無し用 lift_words)

共感・気づき系:
- 「気づいた」「わかった」「実感した」
- 「〜する人の共通点」
- 「意外と」「やってみたら」
- 「◯日目で気づいた罠」

ニュース反応系:
- 【速報】【朗報】【衝撃】【悲報】(CLAUDE.md 準拠)
- 「乗り換え続出」「差が開く一方」

- [ ] ハッシュタグ (#) が含まれていない
- [ ] 40〜220字の範囲
- [ ] 最後の一文が CTA ではなく余韻になっている
